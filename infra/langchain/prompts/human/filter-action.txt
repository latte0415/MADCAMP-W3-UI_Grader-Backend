다음 입력 액션들을 검토하고 처리해줘.

각 액션의 정보:
- action_type: 액션 타입 (fill, click 등)
- action_target: 액션 타겟 설명
- role, name: 요소 식별 정보
- input_type: 입력 필드 타입 (text, email, password 등)
- placeholder: placeholder 텍스트
- selector: CSS selector
- is_filled: 이미 값이 채워져 있는지 여부 (true면 채울 필요 없음)
- current_value: 현재 입력 필드에 채워진 값 (is_filled가 true인 경우)

중요 규칙 (반드시 지켜야 함):
1. is_filled가 true인 액션: 이미 값이 채워져 있으므로 무시해줘 (저장하지도 말고, 최종 답변으로 반환하지도 말아줘).

2. 처리할 수 없는 액션: 메모리에서 적절한 입력값을 찾을 수 없는 액션만 save_action으로 저장해줘.

3. 처리 가능한 액션: 메모리에서 적절한 입력값을 찾을 수 있는 액션은 반드시 final_response로만 반환해줘.
   - 절대 save_action으로 저장하지 말아줘.
   - final_response로 반환할 때는 원본 액션의 모든 필드(action_type, action_target, role, name, selector 등)를 그대로 포함하고, action_value만 적절한 값으로 채워서 반환해줘.

4. 같은 액션을 save_action과 final_response 둘 다 사용하면 안 돼. 하나만 선택해줘.
